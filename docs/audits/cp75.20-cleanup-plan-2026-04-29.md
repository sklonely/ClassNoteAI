# cp75.20+ Cleanup Plan — Post-Phase-7 Stability Pass

**Date**: 2026-04-29
**Scope**: 47 findings from 8 parallel sub-agent audits across the entire ClassNoteAI codebase
**Branch**: `feat/h18-design-snapshot`
**Methodology**: TDD-first — every fix lands with a failing-then-passing regression test

---

## 1 · Why this plan exists

cp75.19 fixed the trash-bin user_id bug. The fix surface was tiny (one component, one frontend call site) but the bug had been silently broken for **weeks** — a cascade of: cp75.3 multi-user migration changed the data owner, sibling code in the same component got the new pattern, but the load path was missed.

This pattern repeats. The 8 audits found **47 independent findings** across multi-user isolation, soft-delete state, recording lifecycle, transcription, summary, chat / RAG, course CRUD, settings, and setup. Most share a small set of root causes. We can fix them in batches with a single round of regression-test scaffolding per cluster.

**Goal**: kill all P0s in 5 commits, cover all P1 transient/race issues in 3 more, leave P2 schema-level work for a separate sprint.

**Anti-regression strategy**:
- Every fix lands with a **new test that fails on the unfixed code**.
- Cross-cutting invariants (e.g. "no read endpoint returns `is_deleted=1` rows") get **shared fixture-driven tests** that exercise every endpoint at once — one place to add the next read endpoint that gets added in the future.
- Each commit message names the audit finding(s) it closes (so future audits can diff against this doc).

---

## 2 · TDD methodology (applied uniformly to every batch)

Every change follows the same loop:

1. **Reproduce** — write a vitest / cargo test that exercises the bug. Test must FAIL on current `main`.
2. **Run** — confirm red. If green, the test isn't actually exercising the bug — go back to step 1.
3. **Fix minimally** — change the smallest amount of code that makes the test green.
4. **Re-run** — confirm green.
5. **Add edge cases** — for the same bug class, write 2-3 nearby tests (boundary values, error paths, concurrent / race scenarios). Each must turn green on the same fix.
6. **Run full suite** — `npx vitest run` and `cargo test --lib` on relevant Rust modules. No regression in unrelated tests.
7. **Commit** — message explicitly references the audit finding ID + lines (`Closes finding 1.1, 1.2, …`).

**Hard rule**: no fix lands without at least one test that demonstrably fails without it. If the test always passes regardless of the fix, the test is wrong.

|---|---|---|

### **cp75.22** — Recording lifecycle cleanup (4 P0s)

**Goal**: subtitle service is a clean slate at every session boundary; pause/resume math is correct; recovery is atomic.

#### Findings closed
| # | File:line | Issue |
|---|---|---|
| 3.1 | `src/services/recordingSessionService.ts:300` | `_doStart()` doesn't clear subtitleService; new session sees stale segments |
| 3.2 | `src/services/recordingSessionService.ts:649` + `subtitleService.ts:160` | `stop()` unsubscribes but doesn't clear `subtitleService.segments[]` |
| 3.3 | `src/services/recordingSessionService.ts:1199` | Pause/resume elapsed = `(now - sessionStartMs)` — doesn't subtract pause durations |
| 3.4 | `src/services/recordingRecoveryService.ts:213` | `recoverTranscript` + `finalize_recording` not transactional; retry on finalize failure double-imports |

#### Test plan (write FIRST)

`src/services/__tests__/recordingSessionService.cleanup.test.ts`:

```ts
it('start() after stop() begins with empty subtitleService.segments', async () => {
    // session 1: push some segments. stop. Verify subtitleService is non-empty.
    // session 2: start. Verify subtitleService.getSegments() === [].
});

it('stop() clears subtitleService segments', async () => { ... });

it('pause/resume cycle preserves recording-time clock (excludes pause)', async () => {
    // mock Date.now. start at t=0. record 60s. pause at t=60. resume at t=120 (60s pause).
    // record another 30s. stop at t=150.
    // expect elapsed === 90 (not 150).
});

it('recoverTranscript is idempotent across retried finalize', async () => {
    // mock: recoverTranscript imports 5 segments, finalize throws.
    // retry: same 5 segments → DB has 5 rows total (not 10).
});
```

#### Fix sketch
- `_doStart()`: call `subtitleService.setSegments([])` BEFORE `attachSubtitleSubscriber()`.
- `stop()`: call `subtitleService.setSegments([])` after `cleanupListeners()`.
- Pause/resume: introduce `totalPausedMs` accumulator on the service. On pause, stamp `pauseStartedAtMs`. On resume, `totalPausedMs += now - pauseStartedAtMs`. Elapsed = `(now - sessionStartMs - totalPausedMs) / 1000`.
- Recovery transactional: wrap `recoverTranscript + finalize_recording` in a Rust-side transaction OR add a checksum / "last imported segment id" marker in the DB so re-import is idempotent.

#### Acceptance criteria
- [ ] 4 new tests pass
- [ ] Existing `recordingSessionService.stop.test.ts` (27 tests) still passes
- [ ] Manual verify: record 30s, pause 5s, resume 30s — clock shows 60s

### **cp75.24** — Translation sidecar lock + Parakeet variant safety (2 P0s)

**Goal**: rapid provider/variant switching can't spawn duplicate sidecar processes or corrupt mid-session inference.

#### Findings closed
| # | File:line | Issue |

| 7.2 | `src-tauri/src/lib.rs:597-628` Parakeet variant switch | Mid-session switch uses old session id; engine buffer mid-inference on old variant |

#### Test plan (write FIRST)

## 5 · Cross-cutting invariant tests (one-time scaffolding)

These tests exercise EVERY endpoint of a class and act as a tripwire for future regressions.

### 5.1 · "no read endpoint returns deleted rows" (Rust)
File: `src-tauri/src/storage/invariant_test.rs`

For each Rust read endpoint that takes an id of a soft-deletable resource, seed the DB with a soft-deleted row, call the endpoint, assert the result is None / empty / Err. The list of endpoints is iterated programmatically via a const slice of (name, fn). When someone adds a new read endpoint, they MUST either add it to the slice or annotate `#[allow(invariant_test_read_endpoints)]`.

### 5.2 · "every write Tauri command requires user_id" (TS)
File: `src/services/__tests__/multiUserInvariant.test.ts`

Iterate over every `invoke('cmd', args)` call in the codebase (parse via tsserver / regex). For each, check args contains `userId` OR the command is in an allowlist of "intentionally global" commands.

### 5.3 · "subtitleService is empty after stop" (TS)
File: `src/services/__tests__/subtitleServiceLifecycle.test.ts`

Drive the recordingSessionService through start → segments emitted → stop. Assert subtitleService.getSegments() === [] after stop. Repeat for pause/resume/stop and start/stop/start cycles.

### 5.4 · "deleted_at is always set when is_deleted=1" (Rust)
For every UPDATE that sets `is_deleted=1`, also assert `deleted_at` is non-NULL. This catches the legacy-trash issue.

## 9 · How to use this doc

When you sit down to work on cp75.20:
1. Open this file.
2. Pick a batch from Section 4.
3. Open the test file (or create it) named in the batch's "Test plan".
4. Write the failing tests first. Don't touch production code.
5. Run them — confirm red.
6. Open the production files named in the batch's "Findings closed" table.
7. Fix until green.
8. Run the full suite.
9. Commit. Move the batch's checkbox in Section 8 to checked.

- Add them to the appropriate batch (or open a new cp75.28+ batch in Section 4).
