## Three Cross-cutting Themes

### Theme 1 вАФ Cancel buttons systematically missing (P0 √Ч 4)
- `H18AIDock` / `H18AIPage` / `AIChatPanel` вАФ `chatStream` AbortController is wired in services but **none of the three entry points expose a "stop generating" button**.
- Gemma / Parakeet model downloads вАФ once started, only path out is error or app close.
- **Fix shape**: AbortController infrastructure already exists; UI just needs to surface it via a shared `<CancelButton>` primitive.

### Theme 2 вАФ Errors fall into `console.warn` blackholes (P0вАУP1 √Ч 15+)
- `H18DeepApp.tsx` вАФ 6 catches that only log, no toast.
- `AIChatPanel.tsx:378-388` вАФ 401/429/5xx all collapse to `гАМйМѓи™§пЉЪmsgгАН` string.
- Canvas `commitPairings` failure вЖТ no retry / no per-row rollback.
- `canvasFeedService.ts:409-413` вАФ 401/403 not branched.
- Several `openUrl().catch(() => {})` swallow without UI.
- **Fix shape**: a `surfaceError(err, { kind, retry?, hint? })` helper that classifies by `LLMErrorKind` / HTTP status and renders a toast with a `Retry` CTA.

### Theme 3 вАФ API key hygiene (P0 √Ч 3)
- `ProfilePanes.tsx:1947-1965` вАФ provider API key uses `<input type="text">`, no mask, no show toggle.
- `ProfilePanes.tsx:1428-1439` вАФ Google Translate key has the same problem.
- Canvas RSS feed URL вАФ not strictly secret but same hygiene gap.
- **Fix shape**: shared `<PSecretInput>` component with mask / show toggle / clear.
